## 1. LLM执行摘要

- 当前任务的目标是把 VS1 专用工作流对外部 `ai4sec_opencti_mcp` FastAPI 服务的依赖改为工作流内节点实现。
- 首要修改对象是 `ai4sec_threat_modeling_agent` 对应的独立工作流资产，不应把变更扩散到统一工作流或其他价值流。
- 现状上，KG 已确认 `ai4sec_threat_modeling_agent` 通过关系 1244 访问 `ai4sec_opencti_mcp`，而 `ai4sec_opencti_mcp` 再访问 OpenCTI Runtime Stack。
- 所有进入和流转的数据仍必须保持 OPENCTI 中的 STIX 2.1 语义，不能因为迁移到节点而引入自定义业务数据通道。
- 必须继续满足“内部 OpenCTI 是唯一情报底座”“DIFY Agent 是统一智能入口”的基线原则。
- 本任务范围只限 `ai4sec_threat_modeling_agent` 工作流；是否保留 1215 给其他链路复用，当前 KG 未给出结论，需人工确认。
- 不允许把同一能力再次落回外部 FastAPI 代理，只是换调用点但保留相同运行边界不符合任务目标。
- 最关键验收条件是：VS1 工作流在不依赖外部 1215 服务的前提下，仍能完成 OpenCTI 取数、结果组织和后续威胁建模输入准备。
- 当前主要风险是 Dify 节点内可直接落地的 OpenCTI 认证、查询编排、结果裁剪方式在 KG 中未被详细建模。

## 2. 已确认事实

- `ai4sec_opencti_mcp`（1215）当前被定义为 FastAPI 服务，负责从仓库根环境加载配置、解析稳定别名到 live OpenCTI 对象标识、查询最小 GraphQL 对象包络、补充本地场景上下文、接收 STIX bundle，并暴露 passthrough webhook 端点。
- `ai4sec_opencti_mcp`（1215）的当前实现仍存在“完整 live STIX projection”和“真实 OpenCTI bundle persistence”未完全落地的描述。
- `ai4sec_threat_modeling_agent`（1436）当前被定义为 VS1 的独立 Dify advanced-chat workflow agent，代码路径是 `DifyAgentWorkflow/ai4sec_threat_modeling_workflow.yaml`。
- 关系 1244 明确表示 `ai4sec_threat_modeling_agent`（1436）当前通过“access opencti stix data”访问 `ai4sec_opencti_mcp`（1215）。
- 关系 1083 明确表示 `ai4sec_opencti_mcp`（1215）访问 `OpenCTI Runtime Stack`（1276）。
- 原则 `1226` 要求业务流程中所有数据都必须是 OPENCTI 平台中的 STIX 2.1 标准数据。
- 原则 `1228` 要求推送到 DIFY Agent 的情报都来自内部 OpenCTI，外部平台只能对接内部 OpenCTI。
- 原则 `1229` 要求主动交互和被动 webhook 调度统一收敛到 DIFY Agent。
- Dify 当前版本中，VS1 工作流允许直接通过原生 HTTP/工具节点安全访问 OpenCTI
- 在本任务完成后，`ai4sec_opencti_mcp`（1215）是否仅对 `ai4sec_threat_modeling_agent` 停用，被其他链路继续保留
- 工作流节点内的 OpenCTI 凭证使用明文，直接从external_opencti\.env中获取OPENCTI_ADMIN_TOKEN
- 节点化后仍需要保留本地 alias 兼容层或部分共享运行时工具
- 需要在应用层显式建模“工作流内 OpenCTI 访问节点”，由人工新增架构元素。

## 4. 约束与边界

- 必须遵守 `Progressive Disclosure`：工作流只暴露完成 VS1 所需的最小数据与动作，不得把底层 OpenCTI 原始查询细节直接暴露给对话层。
- 必须遵守 `Separation of Concerns`：工作流编排、OpenCTI 访问、LLM 分析、下载输出分别落在清晰节点边界内，不得把所有职责混成单一脚本节点。
- 必须遵守原则 `1226`、`1228`、`1229`；如涉及高等级通知也不得绕过 `1230`，但本任务未要求补齐通知分支。
- 必须保持不变的边界：统一工作流 `ai4sec_agent` 主链路、Notification MCP、OpenCTI 运行栈部署拓扑、其他价值流工作流。
- 明确禁止：继续依赖外部 FastAPI 1215 完成 VS1 主链路、绕过 OpenCTI 直接消费外部原始情报、在本任务中扩展到非 VS1 业务流。
- 明确禁止：为了迁移方便而放宽 STIX 2.1 语义、引入与 OpenCTI 不一致的中间数据格式、顺手改造无关容器编排。
