## 9. 风险、阻塞与缓解措施

- 技术风险：OpenCTI 版本 API 差异造成写入失败。缓解：引入版本能力探测和回退策略。
- 数据风险：字段矩阵不完整导致对象被拒收。缓解：先最小可用集，逐步扩展并回归。
- 运维风险：一致性窗口波动超过阈值。缓解：增加异步重试与延迟容忍配置。
- 质量风险：团队误用 mock 结果。缓解：在 CI 中将真实集成验证设为必经关卡。
