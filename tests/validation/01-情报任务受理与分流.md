# Capability 测试用例：情报任务受理与分流

## 1. Capability 定义

- 一级能力域：情报接入与标准化
- 关联架构元素：1208、1222、1223、1224、1225、1226、1228、1229
- 覆盖目标：验证系统能否把新任务、新告警、新漏洞、新设计风险收敛为待处理业务任务。

## 2. 前置条件

- 先完成 [00-environment-setup.md](./00-environment-setup.md) 中的测试环境搭建，并确认 Dify、OpenCTI、OpenCTI MCP、Notification MCP 均可访问。
- 已导入 [00-virtual-test-data.md](./00-virtual-test-data.md) 中 VS1、VS2、VS3、VS4 的 4 组测试数据。
- 测试人员可访问 Dify 与 OpenCTI。
- 使用 `ANALYST_ACCOUNT` 登录主流程，必要时用 `ROLE_SHARED_ACCOUNT` 核验消费结果。

## 4. 补充观察项

- 是否存在统一任务池或统一场景中心。
- 是否能区分主动发起任务与被动 webhook 输入。
- 是否存在重复任务或无法幂等识别的问题。
